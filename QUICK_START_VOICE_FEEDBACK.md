# 음성 피드백 빠른 시작 가이드

## 🚀 5분 안에 시작하기

### 1단계: 환경 변수 설정

`.env.local` 파일에 추가:
```bash
# OpenAI API (필수)
LLM_API_KEY=your-openai-api-key-here
LLM_API_URL=https://api.openai.com/v1/chat/completions

# Firebase (기존 설정 유지)
NEXT_PUBLIC_FIREBASE_API_KEY=...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=...
NEXT_PUBLIC_FIREBASE_PROJECT_ID=...
```

### 2단계: 서버 실행

```bash
npm run dev
```

### 3단계: 음성 피드백 사용

1. 브라우저에서 `http://localhost:3000` 접속
2. 로그인
3. 네비게이션에서 **"🎤 음성 피드백"** 클릭
4. 최초 접속 시 설문 완료:
   - 톤 선택 (친근한/전문적인/격식 있는)
   - 상세도 선택 (간단히/상세히/포괄적으로)
5. "🎤 녹음 시작" 버튼 클릭
6. 고민이나 질문 말하기 (예: "자소서가 너무 추상적인 것 같아요")
7. "⏹️ 녹음 종료" 클릭
8. 요약 확인 후 "계속하기"
9. 맞춤 피드백 확인!

## 📱 사용 예시

### 예시 1: 자소서 고민
**사용자 음성:**
> "제가 쓴 자기소개서가 너무 추상적인 것 같아요. 구체적인 예시를 넣고 싶은데 어떻게 해야 할까요?"

**AI 요약:**
> "자기소개서의 구체성을 높이고 싶어 하시네요"

**AI 피드백:**
1. ✅ **정량적 성과 추가하기**
   "프로젝트를 완수했다" 대신 "5명 팀에서 리드로 3개월간 프로젝트를 진행하여 사용자 만족도 35% 향상"과 같이 구체적 수치를 포함하세요.

2. ✅ **STAR 기법 활용하기**
   상황(Situation) → 과제(Task) → 행동(Action) → 결과(Result) 순서로 경험을 구조화하면 읽는 사람이 명확히 이해할 수 있습니다.

3. ✅ **기술 스택 구체화하기**
   "다양한 언어 사용"이 아니라 "React, TypeScript, Next.js를 활용하여 3개의 상용 서비스 개발"처럼 명확히 명시하세요.

---

### 예시 2: 면접 준비
**사용자 음성:**
> "다음 주에 기술 면접이 있는데, 프로젝트 경험을 어떻게 설명해야 할지 모르겠어요."

**AI 요약:**
> "기술 면접에서 프로젝트 경험을 효과적으로 설명하고 싶어 하시네요"

**AI 피드백:**
1. ✅ **프로젝트 맥락 먼저 설명**
   "왜 이 프로젝트를 시작했나요?"부터 시작하세요. 비즈니스 문제와 기술 과제를 연결하면 면접관이 공감합니다.

2. ✅ **기술적 의사결정 강조**
   "React를 사용했어요" → "React를 선택한 이유는 컴포넌트 재사용성과 빠른 개발 속도 때문이었고, 실제로 개발 기간을 2주 단축했습니다"

3. ✅ **트레이드오프 언급**
   완벽한 선택은 없습니다. "이 방법의 장점은 A이지만, B라는 한계도 있었고, 그래서 C로 보완했다"는 식으로 말하면 깊이가 느껴집니다.

## 🎯 주요 기능

### 1. 🤖 지능형 요약
음성을 듣고 핵심 의도를 한 문장으로 요약합니다.

### 2. 🎨 개인화
- **톤:** 친근한/전문적인/격식 있는
- **상세도:** 간단히/상세히/포괄적으로
- **자동 학습:** 평가에 따라 다음 피드백 자동 개선

### 3. 📊 구조화된 피드백
- **핵심 요약:** 한 문장
- **실행 가능한 조언:** 3가지 (구체적, 실용적)
- **전체 분석:** 숨김 (필요 시 펼치기)

### 4. 💬 대화형 학습
- 👍 유용했어요 → 긍정적 피드백 저장
- 👎 아쉬워요 → 이유 선택 → 다음부터 자동 개선
  - "설명이 추상적이에요"
  - "예시가 더 필요해요"
  - "문장을 다듬어주세요"

## 🔧 문제 해결

### "마이크 권한이 거부되었습니다"
1. 브라우저 주소창 옆 자물쇠 아이콘 클릭
2. "마이크" 권한을 "허용"으로 변경
3. 페이지 새로고침

### "음성 인식이 지원되지 않습니다"
- **Chrome/Edge 브라우저 사용 필수**
- Safari/Firefox는 현재 미지원

### "음성 처리 중 오류가 발생했습니다"
1. 환경 변수 `LLM_API_KEY` 확인
2. OpenAI API 크레딧 확인
3. 네트워크 연결 확인

### "말씀이 잠깐 끊긴 것 같아요"
- 더 길게 말씀해주세요 (최소 5글자 이상)
- 소음이 적은 환경에서 녹음
- 마이크에 가까이 대고 또박또박

## 💡 팁

### 효과적인 질문 방법
✅ **좋은 예:**
- "자소서가 너무 추상적인 것 같아요"
- "공고와 자소서의 연결성을 높이고 싶어요"
- "면접에서 프로젝트 경험을 어떻게 설명해야 할까요?"

❌ **피해야 할 질문:**
- "안녕하세요" (너무 짧음)
- "오늘 날씨가 좋네요" (주제 벗어남)
- "..." (침묵)

### 피드백 품질 높이기
1. **구체적으로 질문하기**
   - 나쁨: "자소서 좀 봐주세요"
   - 좋음: "자소서에서 기술 스택을 어떻게 표현해야 할까요?"

2. **맥락 제공하기**
   - "공고에 React가 필수인데, 제 경험을 어떻게 어필할까요?"

3. **솔직하게 평가하기**
   - 피드백이 추상적이면 👎 "설명이 추상적이에요" 선택
   - AI가 다음부터 자동으로 개선합니다!
